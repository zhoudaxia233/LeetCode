# 解题思路：
>[题目链接](https://leetcode.com/problems/reverse-bits/description/)

这题用Python做相当简单，不需要思路。  
但有一点需要注意：文中强调了是无符号32位整数，因此我们在反转二进制字符串之前，需要在字符串的前面进行zero-padding，也就是填充一堆零，使整个字符串的长度达到32位。否则，当我们将字符串反转之后，得到的数字将是错误的。
>假如我们要反转`1101`，如果不进行zero-padding，我们得到的将是`1011`，而实际上我们想得到的值是`1011 0000 0000 0000 0000 0000 0000 0000`。
